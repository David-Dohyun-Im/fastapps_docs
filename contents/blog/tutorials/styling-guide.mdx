# Complete Styling Guide

In this comprehensive guide, we'll explore how to style your applications beautifully and efficiently using modern CSS practices and Tailwind CSS.

## Getting Started with Tailwind

Our platform comes with Tailwind CSS pre-configured. You can start using utility classes immediately:

```tsx
<div className="flex items-center justify-center p-4 bg-primary text-white rounded-lg">
  <h1 className="text-2xl font-bold">Hello World</h1>
</div>
```

## Custom Themes

### Creating a Theme

Define your custom theme in `styles/globals.css`:

```css
:root {
  --color-primary: oklch(0.488 0.243 264.376);
  --color-secondary: oklch(0.696 0.17 162.48);
  --radius-lg: 0.625rem;
}
```

### Using Theme Variables

Access your theme variables in components:

```tsx
<button className="bg-primary text-primary-foreground rounded-lg">
  Custom Button
</button>
```

## Dark Mode Support

### Automatic Dark Mode

Enable dark mode with a simple toggle:

```tsx
import { ThemeToggle } from '@/components/theme-toggle'

export function Header() {
  return (
    <header>
      <ThemeToggle />
    </header>
  )
}
```

### Dark Mode Styles

Style for dark mode using the `dark:` prefix:

```tsx
<div className="bg-white dark:bg-gray-900 text-black dark:text-white">
  This adapts to the theme
</div>
```

## Responsive Design

### Breakpoint System

Use responsive utilities for different screen sizes:

```tsx
<div className="grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3 gap-4">
  {items.map(item => (
    <Card key={item.id} {...item} />
  ))}
</div>
```

Breakpoints:
- `sm`: 640px
- `md`: 768px
- `lg`: 1024px
- `xl`: 1280px
- `2xl`: 1536px

## Component Patterns

### Card Component

Create reusable styled components:

```tsx
export function Card({ children, variant = 'default' }) {
  return (
    <div className={cn(
      "rounded-lg border p-6",
      variant === 'default' && "bg-card",
      variant === 'outlined' && "border-2"
    )}>
      {children}
    </div>
  )
}
```

### Button Variants

Use the built-in button component:

```tsx
import { Button } from '@/components/ui/button'

<Button variant="default">Primary</Button>
<Button variant="secondary">Secondary</Button>
<Button variant="outline">Outline</Button>
<Button variant="ghost">Ghost</Button>
```

## Animation and Transitions

### Hover Effects

Add smooth transitions:

```tsx
<button className="transition-all hover:scale-105 hover:shadow-lg">
  Hover me
</button>
```

### Keyframe Animations

Use Tailwind's animation utilities:

```tsx
<div className="animate-bounce">
  Bouncing element
</div>

<div className="animate-pulse">
  Pulsing element
</div>
```

## Best Practices

### 1. Use Semantic Colors

Instead of hardcoding colors, use semantic variables:

```tsx
// ❌ Don't do this
<button className="bg-blue-500 text-white">

// ✅ Do this
<button className="bg-primary text-primary-foreground">
```

### 2. Consistent Spacing

Use the spacing scale consistently:

```tsx
// Spacing: 0, 0.5, 1, 2, 3, 4, 6, 8, 12, 16, 20, 24
<div className="p-4 space-y-6">
  <div className="mb-8">Section 1</div>
  <div className="mb-8">Section 2</div>
</div>
```

### 3. Mobile-First Design

Start with mobile styles, then add larger breakpoints:

```tsx
<div className="text-sm md:text-base lg:text-lg">
  Responsive text
</div>
```

### 4. Accessibility

Always consider accessibility:

```tsx
<button 
  className="focus:outline-none focus:ring-2 focus:ring-primary"
  aria-label="Close menu"
>
  Close
</button>
```

## Advanced Techniques

### CSS Grid Layouts

Create complex layouts with Grid:

```tsx
<div className="grid grid-cols-12 gap-4">
  <div className="col-span-12 lg:col-span-8">
    Main content
  </div>
  <div className="col-span-12 lg:col-span-4">
    Sidebar
  </div>
</div>
```

### Custom CSS

When you need custom CSS:

```css
/* styles/custom.css */
.gradient-text {
  @apply bg-gradient-to-r from-primary to-secondary;
  -webkit-background-clip: text;
  -webkit-text-fill-color: transparent;
}
```

## Performance Tips

1. **Purge unused CSS**: Tailwind automatically removes unused styles in production
2. **Avoid inline styles**: Use Tailwind classes instead
3. **Use CSS variables**: For dynamic theming
4. **Optimize animations**: Use `transform` and `opacity` for better performance

## Conclusion

Styling your application doesn't have to be complicated. With Tailwind CSS and our design system, you have everything you need to create beautiful, responsive, and accessible interfaces.

Experiment with different combinations and find what works best for your design!
